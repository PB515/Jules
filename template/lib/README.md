# lib/ — day-1 library

*The proven helpers and patterns every build reinvented. Built in Slice 2; typechecks + builds green.*

- `site.ts` — brand/contact constants, set once per site (the gap that forced a multi-file rebrand on Bugadi). **Per-site 20% — edit on clone.**
- `icons.ts` — single icon re-export point (lucide-react), swappable in one place.
- `security.ts` — form honeypot (`HONEYPOT_FIELD`, `failsHoneypot`) + in-memory `rateLimit` + `clientIp`.
- `supabase/` — the **4-client split**: `client` (browser) · `server` (RSC/actions) · `middleware` (session refresh) · `service-role` (server-only, bypasses RLS). Plus `database.types.ts`, generated by `tooling/migrate`.
- `integrations/` — adapter-boundary stubs: `payments/`, `shipping/` (filled by the billing/PWA modules, Tier 2).
- `patterns/` — `empty-state` · `chip-list-editor` · `view-edit-form` · `audit-log` · `two-query-write` · `has_role.sql`. Each carries a one-line "use when" header; see `patterns/README.md`.

Env vars: copy `.env.example` → `.env.local`. The Supabase clients read `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`, and (service-role only) `SUPABASE_SERVICE_ROLE_KEY`.
